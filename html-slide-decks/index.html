The Origins of Slideshows

  • In 1980 developers started to work on early prototypes of presentation software.
  • Intended for artists to design slides for projectors, which had to be printed.
  • Forethought, Inc. changed that in 1987 with the release of PowerPoint for McIntosh.

Web Basics

  • Hyper Text Markup Language (HTML)
    • is a markup language that defines the building blocks that make up a website.
  • Cascading Style Sheets (CSS)
    • makes it possible to style those elements dynamically and change where and how they are displayed on thescreen.
  • JavaScript
    • is a programming language that can modify the behaviour of a website.
  • HTML defines the content of a website, CSS defines the layout of that content and JavaScript adds interactivity.

Slidy & Slidy2

  • Developed by Dave Raggett in 2005.
  • One of the first HTML based presentation frameworks.
  • Slides are defined in blocks <div class="slide">...</div>.
  • Includes: show.css, slidy.js, print.css (optional)
  • Example

S5 & S6

  • Stands for Simple Standards-based Slide Show System.
  • Developed in 2015 by Eric Meyer.
  • Similar to Slidy.
  • Example

Diascope

  • Developed by Martin Stoll in 2006.
  • Diascope comes in the form of a binary executable.
  • The output of diascope is an MPEG encoded video file ready for burning onto a DVD.
  • The input is a list of instructions for rendering.
  • 
  create 2 white title=0,1,1,0 "Diascope\nSlideshow\nGenerator"
  trns 0.3 luma dir=ro
  set font=ArialB fill=gray stroke=white strokewidth=1

  create 0.8 blue title=0.8 "diascope.sf.net"
  trns 0.4 luma dir=td sharp=1
  create 0.8 red title=0.8 "diascope.sf.net"
  trns 0.4 luma dir=td sharp=1
  create 0.8 yellow title=0.8 "diascope.sf.net"
  trns 0.4 luma dir=td sharp=1
  create 0.8 blue title=0.8 "diascope.sf.net"

  trns 0.3 luma dir=ri
  create 1 black

Adobe Flash

  • Used to be omnipresent on the web following its release in 1996.
  • Flash is a powerful tool for creating standalone interactive multimedia applications that can be embedded into a web site.
  • Supports keyboard and mouse.
  • Steve Jobs went over the shortcomings of Flash in a blogpost titled "Thoughts on Flash":
  • "Flash was created during the PC era – for PCs and mice. Flash is a successful business for Adobe, and we can understand why they want to push it beyond PCs. But the mobile era is about low power devices, touch interfaces and open web standards – all areas where Flash falls short. [...] New open standards created in the mobile era, such as HTML5, will win on mobile devices (and PCs too)."
  • Flash will be officially retired in 2020.

The Four Main Classes of Modern Slide Decks

  • Text-based slide decks.
  • JavaScript-based slide decks.
  • Hosted slide decks.
  • Responsive slide decks.

Text-based Slide Decks

  • Do not require the slide creator to possess any knowledge about scripting or styling their slides.
  • Either use pure plaintext or Markdown.
  • The main advantage of this approach is the fast and easy way to write and read slides.
  • The trade-off is less flexibility and modifiability.

Slide Show (S9).

  • Command-line tool developed by Gerald Bauer
  • Takes a Markdown file as input.
  • Turns it into a number of different slide decks, like Slidy, S5, S6.
  • 
$ slideshow build slides.text -t s6blank

=> Preparing slideshow ’slides.html’...
=> Done.
  • Possibility to enhance with ruby scripts.

Slidifier

  • Developed by Holger Ludvigsen.
  • Defines its own plaintext syntax for certain features.
  • Example

Remark

  • Developed by Ole Petter Bang.
  • Uses a mixture of HTML and Markdown.
  • Extends the Markdown syntax with properties such as:
  • name, class, background-image, count, template, layout, exclude.
  • Example

JavaScript-based Slide Decks

  • Heavy focus on scripting.
  • Wide range of functionality.
  • They often make use of third party packages (jQuery).
  • Many of them can be installed using Node.js and npm.

Google I/O Slides

  • Is an HTML template developed for Google I/O.
  • Started development under the name html5slides in 2011.
  • Makes use of external tools such as compass, a ruby tool for converting SASS into CSS styling, and python to convert markdown into HTML.
  • Development has been stopped in 2015. (likely in favor of Google Slides)
  • Example

Fathom.js

  • Developed by Mark Dalgleish.
  • Makes use of jQuery Mobile to optimize for touch controls.
  • Uses the events: tap, swipe left, swipe right, scroll start, scroll stop and orientation change.
  • Can sync a slideshow to an external video.
  • 
$('#presentation').fathom({
  timeline: [ 0, 5, 20, 30, 40, 50, '1:00', '1:15', 90, 120, 155 ],
  video: {
    source: 'vimeo',
    id: '16917950',
    parent: '#vimeo',
    autoplay: true
  }
});

Deck.js

  • Developed by Caleb Troughton.
  • Is based on jQuery and Modernizr.
  • Uses extensions to allow slide creators to create customized presentations that fit their needs.
  • Example

DZSlides

  • Developed by Paul Rouget.
  • Is a single ready to use HTML template.
  • Uses so called shells to further enhance the presentation.
  • In order for these extensions to function, DZSlides has implemented a messaging system that includes:
  • Messages for navigation, getting and setting flags and messages that bind new event listeners.
  • Example

Bespoke.js

  • Developed by Mark Dalgleish.
  • Super minimal (1KB), the main package only provides
    basic features and an API, everything else is managed via plugins.
  • Makes use of a Generator that automatically imports the needed plugins.
  • 
  var bespoke = require('bespoke'),
  cube = require('bespoke-theme-cube'),
  keys = require('bespoke-keys'),
  touch = require('bespoke-touch');

  var deck = bespoke.from('#presentation', [
    cube(),
    keys(),
    touch()
  ]);

Inspire.js

  • Formerly known as CSSS. (up until September 2018)
  • Developed by Lea Verou.
  • Lean and puts a focus on reusability.
  • Many unique features:
  • Video annotations, live code examples, reuse a slide by its ID.
  • Example

Reveal.js

  • Developed by Hakim El Hattab.
  • Relies on many third party tools but is the most feature rich.
  • Vertically nested slides, markdown content, keyboard and touch controls, pdf export, speaker notes, presenter mode, math functions, lazy loading.
  • Highly configurable as well: rtl, autoSlide, history.
  • Has an online editor at: slides.com.
  • Example

Impress.js

  • Developed by Bartek Szopka and Henrik Ingo.
  • Puts a focus on stunning transitions and animations, which are made possible by CSS3 transformations.
  • The entire presentation is set up on one infinite canvas that is traversed by a camera. (Similar to Prezi)
  • 
  <div class="step" data-x="6200" data-y="4300" data-z="-100"
  data-rotate-x="-40" data-rotate-y="10" data-scale="2">
    <p>one more thing...</p>
  </div>
  • Example

Hosted Slide Decks

  • Are neatly packed into a web application.
  • Also serve the tools for creating slides.
  • Everything is in the cloud!

Google Slides

  • Are a part of Google's free online office suite.
  • Can export and import to and from PowerPoint files.
  • Multiple slide creators can edit a presentation simultaneously.

Prezi

  • Originally developed as an app for iOS, Android and Flash.
  • An HTML5 based hosted version was released in 2017 titled Prezi Next.
  • Acquired Infogram, a data visualization platform.
    This allowed them to include tools to:
    • Graph charts and figures.
    • Analyse how much time viewers spend on each slide, how many people viewed the presentation and how many of them shared it.

Showoff

  • Don’t just present; interact with your audience!
  • It supports markdown, live code execution, speaker notes and a presenter mode.
  • Unlike traditional slide decks, Showoff also includes live quizzes, polls and feedback questions.

Responsive Slide Decks

  • Ethan Marcotte describes the main three pillars of Responsive Web Design as flexible grids, fluid images and media queries.
  • Makes it possible to respond to changes in the browser environment and adjust how a web site or application gets displayed accordingly.
  • Can run on a multitude of display sizes and sniff for features rather than requiring them.

Stack

  • Developed by Mike Bostock.
  • Stacks slides vertically rather than going through them horizontally.
  • The primary method of navigation is scrolling, which automatically handles touch support.
  • Adjusts the scaling of slides automatically, but keeps a constant aspect ratio.
  • Activate and deactivate functions can be attached on slide transitions.
  • Example

Scrolldeck.js

  • Developed by John Polacek.
  • Enhances the idea behind stack by using jQuery and Scrollorama.
  • Adds support for animations, incremental lists, an optional sticky navigation header and improves the resizing of slides.
  • Example

Conclusion

  • More open, creative and varied approaches are starting to take the spotlight.
  • In the future, there will be an even greater focus on collaboration and sharing, as can be seen by the most popular tools in use right now.
  • Although they are not perfect by any means, slide decks built for the web are a viable alternative for creating state-of-the-art slideshows.

Rslidy

  • A responsive slide deck built upon Slidy and Slidy2.
  • Responsive HTML5, CSS3, JavaScript (TypeScript). No other dependencies.
  • Now in its fourth iteration.
